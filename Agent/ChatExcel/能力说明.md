# ChatExcel 能力说明

> 面向产品经理及测试人员，描述系统当前支持的核心能力与使用约束。

---

## 一、系统概述

ChatExcel 是一个基于大模型的数据分析工具。用户上传 Excel/CSV 文件，用自然语言提问，系统自动分析数据并返回结果。

系统提供两种分析模式：

| 模式 | 名称 | 核心方式 | 适用场景 |
|------|------|----------|----------|
| **ChatExcel（SQL 模式）** | Text2SQL | 大模型生成 DuckDB SQL 查询数据 | 标准表格的快速查询、统计、聚合 |
| **ReAct Agent（Agent 模式）** | 代码分析 | 大模型生成 Python/Pandas 代码分析数据 | 复杂分析、多步骤任务、生成报告 |

---

## 二、通用能力

### 2.1 文件上传

- **支持格式**：`.xlsx`、`.xls`、`.csv`、`.json`、`.parquet`
- **大小限制**：单个文件不超过 50MB（可通过后端 `.env` 文件中 `MAX_UPLOAD_SIZE_MB` 调整）
- **多文件上传**：同一会话内可上传多个文件，系统自动识别各文件及其 Sheet
- **多 Sheet 支持**：Excel 文件中每个非空 Sheet 会被自动识别为独立数据表
- **文件管理**：每个会话的文件独立存储，删除会话时自动清理对应文件

### 2.2 对话能力

- **流式输出**：问答过程实时流式返回，用户无需等待完整结果
- **历史记录**：对话自动保存，切换会话再切回可恢复历史
- **多轮对话**：支持基于上下文的多轮追问
- **会话管理**：支持新建、重命名、删除会话

### 2.3 大模型配置

- 支持配置多个大模型服务，调用失败时自动切换备用模型
- 当前配置了 3 个模型服务用于容灾切换

---

## 三、ChatExcel 模式（Text2SQL）

> 上传文件后，大模型自动学习表结构（列名翻译、字段含义分析），然后根据用户问题生成 DuckDB SQL 进行数据查询。

### 3.1 数据学习

- 上传文件后自动触发，无需用户操作
- 大模型分析每列的含义，将中文列名翻译为英文标准列名
- 生成表结构描述和数据分析建议，展示给用户

### 3.2 查询能力

| 能力 | 说明 |
|------|------|
| **单表查询** | 筛选、排序、分页、去重 |
| **聚合统计** | COUNT、SUM、AVG、MAX、MIN 等聚合函数 |
| **分组聚合** | GROUP BY 分组统计 |
| **多表连接** | 上传多个文件后，可跨表 JOIN 关联分析 |
| **子查询/CTE** | 支持子查询和 WITH 语句进行复杂查询 |
| **时间函数** | 支持时间戳处理、日期格式转换 |

### 3.3 结果展示

- **自动选择图表类型**：大模型根据问题自动判断最合适的展示方式
- **支持的图表类型**：

| 图表类型 | 适用场景 |
|----------|----------|
| 表格（response_table） | 多列数据展示、非数值列展示（默认类型） |
| 折线图（response_line_chart） | 趋势分析 |
| 饼图（response_pie_chart） | 占比、分布统计 |
| 散点图（response_scatter_chart） | 变量关系探索、异常值检测 |
| 气泡图（response_bubble_chart） | 多变量关系分析 |
| 环形图（response_donut_chart） | 层级结构、分类占比 |
| 面积图（response_area_chart） | 时间序列变化趋势 |
| 热力图（response_heatmap） | 大规模数据分布可视化 |

- **SQL 展示**：查询结果同时展示模型生成的 SQL 语句，便于验证
- **数据预览**：右侧收缩栏可预览上传文件的原始数据

### 3.4 使用约束

- 主要针对**标准表格数据**（规整的行列结构）
- 合并单元格、多表混排等非标准格式可能导致数据丢失
- 大模型生成的 SQL 依赖表结构学习质量，列名不规范时可能影响准确性

## 四、ReAct Agent 模式（代码分析）

> 大模型通过 ReAct（推理-行动）循环自主完成数据分析任务，生成 Python/Pandas 代码执行，支持多步骤复杂任务。

### 4.1 核心分析能力

| 能力 | 说明 |
|------|------|
| **数据探索** | 自动读取文件结构和样本数据 |
| **Pandas 代码分析** | 生成 Python 代码进行数据清洗、统计、分析 |
| **多文件/多表关联** | 支持多文件上传后的跨表分析 |
| **函数聚合/分组统计** | 支持各类聚合、分组、透视分析 |
| **数据可视化** | 生成 matplotlib 图表 |

### 4.2 内置工具集（11 个）

Agent 在分析过程中可自主选择调用以下工具：

| 工具 | 用途 |
|------|------|
| **execute_analysis** | 快速分析文件概况（行列数、字段类型、样本数据） |
| **code_interpreter** | 执行任意 Python 代码，进行数据处理和分析 |
| **html_interpreter** | 将分析结果渲染为 HTML 报告 |
| **load_file** | 加载上传文件信息 |
| **select_skill** | 根据用户问题匹配最相关的分析技能 |
| **load_skill** | 加载分析技能的详细说明 |
| **get_skill_resource** | 读取技能中的参考文档 |
| **execute_skill_script** | 执行技能内联脚本 |
| **execute_skill_script_file** | 执行技能目录下的脚本文件 |
| **todowrite** | 创建和管理任务计划，用于复杂任务拆解 |
| **terminate** | 任务完成，返回最终结论 |

### 4.3 内置分析技能

- 系统内置 **csv-data-analysis** 技能，用于数据探索分析
- 大模型根据用户问题自主判断是否需要使用该技能
- 技能包含参考文档和分析脚本，可扩展

### 4.4 复杂任务支持

| 能力 | 说明 |
|------|------|
| **任务拆解** | 复杂任务（3 步以上）自动拆解为子任务，逐步执行并跟踪进度 |
| **多轮推理** | ReAct 循环（Thought → Action → Observation），最多 30 轮 |
| **上下文管理** | 自动管理上下文窗口，超限时自动压缩历史消息 |
| **错误自愈** | 代码执行失败时，大模型可自动修复并重试 |

### 4.5 HTML 报告生成

- 大模型根据分析结果自主判断是否需要生成 HTML 报告
- 报告可包含数据表格、图表、文字结论
- 报告在前端右侧面板直接渲染展示
- 支持图片插入（代码生成的图表自动转为图片引用）

### 4.6 使用约束

- 每次代码执行相互独立，变量不跨调用保持
- 代码执行有超时限制（默认 60 秒）
- 输出有长度限制（默认 2000 字符，超长截断）
- 复杂任务拆解会增加轮次，响应时间较长

## 五、两种模式对比

| 维度 | ChatExcel（SQL 模式） | ReAct Agent（Agent 模式） |
|------|----------------------|--------------------------|
| **分析方式** | 生成 SQL 查询 DuckDB | 生成 Python 代码执行 |
| **响应速度** | 快（单次 SQL 查询） | 较慢（多轮 ReAct 循环） |
| **适合场景** | 标准查询、统计、聚合 | 复杂分析、多步骤任务、报告生成 |
| **多表关联** | SQL JOIN | Pandas merge/concat |
| **结果展示** | 表格 + 图表 + SQL | 代码 + 表格 + 图表 + HTML 报告 |
| **任务复杂度** | 单轮问答 | 多轮推理，支持任务拆解 |
| **数据限制** | DuckDB SQL 语法范围 | Python/Pandas 能力范围 |

## 六、配置项说明

以下配置均通过后端 `.env` 文件修改，修改后重启服务生效：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `MAX_UPLOAD_SIZE_MB` | 50 | 单个文件上传大小上限（MB） |
| `LLM_RETRY_COUNT` | 3 | 大模型调用最大尝试次数（含首次） |
| `LLM_NON_STREAM_TIMEOUT` | 60 | 非流式调用超时（秒） |
| `LLM_STREAM_FIRST_CHUNK_TIMEOUT` | 10 | 流式调用首字超时（秒） |
| `SANDBOX_TIMEOUT` | 60 | Agent 代码执行超时（秒） |
| `SANDBOX_MAX_MEMORY_MB` | 512 | Agent 代码执行内存上限（MB） |
| `CONTEXT_MAX_TOKENS` | 120000 | Agent 上下文窗口最大 token 数 |
| `REACT_MAX_RETRY_COUNT` | 30 | Agent 单次问题最大循环轮数 |
| `CHAT_EXCEL_HISTORY_ROUNDS` | 10 | ChatExcel 历史对话轮数 |
| `REACT_HISTORY_ROUNDS` | 10 | Agent 历史对话轮数 |
