# AI日报 项目梳理

## 1. 项目一句话定位

基于 FastAPI 的 AI 新闻热点自动分析系统，通过多源搜索、智能筛选、内容聚合和趋势分析，每日自动生成 AI 行业热点报告并支持邮件推送。

## 2. 这个代码仓解决什么问题

### 痛点

- AI 行业信息碎片化：新闻散落在数十个站点（AIBase、机器之心、OpenAI 官网等），人工逐站浏览耗时巨大
- 信息噪声多：搜索结果中混有大量传统 AI 技术（深度学习、机器学习等）和过期内容，难以快速定位大模型领域热点
- 缺乏趋势洞察：单日新闻只看到事件，无法看到一周内的趋势变化和关联

### 处理思路

项目通过一个 10 步的端到端流水线，自动化完成"搜索 → 筛选 → 抓取 → 压缩 → 总结 → 趋势分析"全过程：

1. 从预设信源 URL 抓取内容，按日期过滤
2. 基于查询模板迭代生成搜索关键词
3. 多搜索引擎并发搜索
4. LLM 置信度打分筛选高相关链接
5. 并发抓取网页全文
6. LLM 压缩提炼关键信息
7. LLM 生成今日新闻总结
8. 保存到 SQLite 数据库
9. 读取过去 7 天历史数据
10. LLM 生成趋势对比分析报告

## 3. 技术架构

### 3.1 整体架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| Web 框架 | FastAPI + Uvicorn | 提供 HTTP API，支持 SSL |
| 大模型调用 | LangChain BaseChatModel 自定义实现 | 兼容 OpenAI 接口，支持流式/非流式/异步，记录 token 用量到 CSV |
| 搜索引擎 | DuckDuckGo（ddgs）+ 火山 Coze 工作流 | 双引擎搜索，火山为主力 |
| 网页抓取 | Coze 工作流 + Jina Reader API | 双通道抓取，Coze 为主力 |
| 数据库 | SQLite | 轻量存储每日报告 |
| 邮件推送 | SMTP（QQ 邮箱） | 将报告转为 HTML 发送 |
| 配置管理 | python-dotenv | .env 文件管理密钥和配置 |

## 4. 端到端业务流程

```mermaid
flowchart LR
    A[读取预设信源 URL] --> B[抓取信源内容并按日期过滤]
    B --> C[迭代生成搜索 Query]
    C --> D[多引擎并发搜索]
    D --> E[LLM 置信度打分筛选]
    E --> F[并发抓取网页全文]
    F --> G[LLM 批量压缩内容]
    G --> H[LLM 生成今日报告]
    H --> I[保存到 SQLite]
    I --> J[读取过去7天历史]
    J --> K[LLM 趋势对比分析]
    K --> L[返回最终结果]
```

### 流程步骤

#### 第一步：预设信源处理

读取 `reference/news-url.txt` 中的 105 个预设 URL，抓取内容后用 LLM 过滤出当天或昨天的新闻，并生成摘要。

#### 第二步：迭代生成搜索查询

读取 `reference/news-query.txt` 查询模板，结合日期范围和关注重点，调用 LLM 迭代 3 次生成搜索 Query（每次约 50 个），去重后得到最终查询列表。

#### 第三步：多引擎搜索

将查询分批（每批 10 个），通过火山搜索引擎并发搜索，去重后得到搜索结果列表。

#### 第四步：置信度筛选

对搜索结果分批（每批 50 个），调用 LLM 对每个结果进行置信度打分（0-1），过滤掉低于 0.7 阈值的结果。

#### 第五步：抓取网页全文

对筛选后的链接分批（每批 50 个），通过 Coze 工作流并发抓取网页全文。

#### 第六步：压缩内容

对网页内容分批（每批 50 个），调用 LLM 压缩提炼关键信息，过滤掉日期范围外的内容。

#### 第七步：生成今日报告

将压缩内容和信源摘要合并，调用 LLM 生成结构化的今日 AI 新闻总结（新闻速览 + 深度分析）。

#### 第八步：持久化

将今日报告保存到 SQLite 数据库 `db/ai_today_news.db` 的 `today_news` 表。

#### 第九步：读取历史

从数据库读取过去 7 天的历史报告。

#### 第十步：趋势分析

将今日报告与历史数据合并，调用 LLM 生成趋势对比分析报告，最终拼接返回。

## 6. 核心类 TodayNews 详细流程

**定义位置**：[workflow/today_news.py](file:///d:/code/prompt-engineering/Agent/AI日报/workflow/today_news.py) → `TodayNews`

### 6.1 初始化参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `search_iter_num` | 3 | 搜索 Query 迭代生成次数 |
| `confidence_threshold` | 0.7 | 置信度筛选阈值 |
| `compress_num` | 50 | 内容压缩批次大小 |
| `fetch_url_batch` | 50 | URL 抓取批次大小 |
| `llm` | — | ChatOpenAIModel 实例，从环境变量初始化 |

### 6.2 `get_today_news()` — 主流程

依次调用以下方法，构成完整流水线：

| 步骤 | 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|------|
| 1 | `fetch_reference_url()` | news-url.txt | reference_summaries (str) | 预设信源抓取+日期过滤+摘要 |
| 2 | `generate_query()` | news-query.txt | querys (list[str]) | 迭代3次生成搜索Query |
| 3 | `search_query(querys)` | querys | query_link (list[dict]) | 火山搜索+去重 |
| 4 | `select_link(query_link)` | query_link | selected_urls (list[str]) | LLM置信度打分+阈值过滤 |
| 5 | `fetch_links(links)` | links | all_content (list[dict]) | Coze并发抓取网页全文 |
| 6 | `compress_content(all_content)` | all_content | compressed_results (list[str]) | LLM批量压缩提炼 |
| 7 | `summary_today(compressed, ref)` | 压缩结果+信源摘要 | today_results (str) | LLM生成今日报告 |
| 8 | `save_to_db(today_results)` | 今日报告 | — | SQLite持久化 |
| 9 | `read_last_week_news()` | — | last_week_news (str) | 读取7天历史 |
| 10 | `analyze_last_week_news(today, last)` | 今日+历史 | final_result (str) | LLM趋势分析 |

### 6.3 `fetch_reference_url()` — 预置信源处理

1. 读取 `reference/news-url.txt`，获取 105 个预设 URL
2. 调用 `fetch_links()` 抓取全部网页内容
3. 调用 `filter_by_date()` 进行日期过滤：
   - 分批（50个/批），调用 LLM 判断每条内容的发布日期
   - 使用 `filter_date_prompt` 提示词，要求只保留今天或昨天的内容
   - 后处理 `_post_filter_by_date()` 用正则提取日期，二次过滤确保准确性
4. 返回过滤后的摘要文本

### 6.4 `generate_query()` — 迭代生成搜索查询

1. 读取 `reference/news-query.txt` 查询模板
2. 计算日期范围（过去 24 小时）
3. 迭代 `search_iter_num`（默认 3）次：
   - 调用 `generate_querys()` 生成一批查询
   - 将已生成的查询拼接为上下文 `existing`，传入下次迭代避免重复
   - 每次生成约 50 条查询
4. 全局去重后返回查询列表

### 6.5 `search_query()` — 多引擎搜索

1. 将查询分批（10 个/批）
2. 每批通过 `asyncio.to_thread` 并发调用 `huoshan_search()`
3. 每批之间 `sleep(5)` 减缓压力
4. 按 URL 去重后返回结果

### 6.6 `select_link()` — 置信度筛选

1. 将搜索结果分批（50 个/批）
2. 为每个 URL 生成 `url_1`, `url_2`... 形式的 ID 映射
3. 调用 LLM（`select_link_prompt`）对每条结果打分（0-1），评分维度：内容相关性(0.4)、技术热点性(0.3)、时效性(0.2)、内容质量(0.1)
4. 解析 LLM 返回的 JSON，提取 ID、置信度、理由
5. 过滤掉置信度 < `confidence_threshold`（0.7）的链接
6. 将筛选后的 URL ID 映射回原始 URL

### 6.7 `fetch_links()` — 抓取网页全文

1. 将链接分批（50 个/批）
2. 通过 `asyncio.to_thread` 并发调用 `coze_fetch()` 抓取
3. 过滤掉空结果和 "null" 内容
4. 返回 `[{url, content}]` 列表

### 6.8 `compress_content()` — 批量压缩

1. 计算日期范围
2. 分批（50 个/批），将网页内容拼接
3. 调用 LLM（`compress_content_prompt`），要求：
   - 严格基于原文，不编造
   - 保留关键信息（产品发布、技术突破、数据指标、人物机构、时间）
   - 去除广告、导航栏等噪声
   - 日期范围外的内容跳过
4. 收集所有压缩结果

### 6.9 `summary_today()` — 生成今日报告

1. 将信源摘要和压缩结果拼接
2. 调用 LLM（`summary_today_prompt`），生成结构化报告：
   - **新闻速览**：每条包含标题、内容、时间、数据
   - **深度分析**：核心趋势、关键洞察、关联分析、机会与挑战、思考问题

### 6.10 `save_to_db()` — 持久化

1. 创建 `db/` 目录
2. 连接 SQLite 数据库 `db/ai_today_news.db`
3. 确保 `today_news` 表存在（字段：id, created_at, content）
4. 插入今日报告记录

### 6.11 `read_last_week_news()` — 读取历史

1. 连接 SQLite 数据库
2. 查询过去 7 天的记录（`date(created_at) >= date(?) AND date(created_at) <= date(?)`）
3. 拼接为 `"日期：xxx\n内容\n======="` 格式的文本

### 6.12 `analyze_last_week_news()` — 趋势分析

1. 将历史数据填入 `analyze_last_week_prompt`
2. 调用 LLM 生成趋势分析报告，包含：
   - **关键事件关联**：技术演进脉络、产品迭代关联、生态联动、趋势延续
   - **深度分析**：核心趋势、关键洞察
   - **机会分析**：行业机会、个人机会
   - **总结**：核心趋势、关键机会、行动建议、思考问题
3. 拼接今日报告 + 趋势分析作为最终结果

## 7. 工具模块详细说明

#### `jina_fetch(url)`

- 使用 Jina Reader API（`https://r.jina.ai/{url}`）抓取网页

### 7.3 LLM 工具 — [utils/llm_generation.py](file:///d:/code/prompt-engineering/Agent/AI日报/utils/llm_generation.py)

#### `call_llm_with_retry(llm, prompt, logger, max_retries=3)`

- 封装 LLM 调用，带重试机制（最多 3 次）
- 智谱速率限制错误（1302）等待 60 秒后重试
- 其他错误等待 10 秒后重试
- 3 次都失败返回 `SimpleNamespace(content="大模型error")`

### 7.4 日志工具 — [utils/logger.py](file:///d:/code/prompt-engineering/Agent/AI日报/utils/logger.py)

#### `define_log_level(log_name, print_level, logfile_level)`

- 创建独立 Logger，控制台 INFO 级别，文件 DEBUG 级别
- 日志格式：`时间 - 级别 - [文件:函数:行号] - 消息`
- 日志文件保存在 `logs/{log_name}.log`

### 7.5 自定义 ChatModel — [llm/ChatOpenAIModel_LangChian.py](file:///d:/code/prompt-engineering/Agent/AI日报/llm/ChatOpenAIModel_LangChian.py)

## 10. 提示词体系

所有提示词定义在 [prompts/today_news.py](file:///d:/code/prompt-engineering/Agent/AI日报/prompts/today_news.py)，核心设计原则：

| 提示词 | 用途 | 关键设计 |
|--------|------|---------|
| `search_url_create` | 生成搜索查询 | 指定8大关注领域 + 3层搜索策略（聚合/技术/补充） + 不关注传统AI |
| `select_link_prompt` | 置信度打分 | 4维评分（相关性0.4/热点性0.3/时效性0.2/质量0.1），5档评分区间，JSON输出 |
| `compress_content_prompt` | 压缩网页内容 | 严格不编造 + 保留5类关键信息 + 去噪声 + 日期范围外跳过 |
| `summary_today_prompt` | 生成今日报告 | 新闻速览 + 深度分析（趋势/洞察/关联/机会/思考） |
| `filter_date_prompt` | 日期过滤 | 严格只保留今天或昨天，输出格式化摘要 |
