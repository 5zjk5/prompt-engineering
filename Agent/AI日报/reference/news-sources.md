## WebSearch 关键词模板

将 `{date_range}` 替换为目标时间描述（如 "today"、"this week"、"March 2026"、"2026-02-20..2026-03-06"）。

> ⚠️ **频率限制提醒**：DuckDuckGo 搜索有频率限制，不要连续发起大量搜索请求。
> 下方提供了「精简版」和「完整版」两套查询模板。**默认使用精简版**（总搜索次数更少），
> 仅在精简版信息不足时才从完整版中挑选补充查询。
> 推荐节奏：每做 2-3 次搜索后，穿插 2-3 次 fetch 抓取操作，再继续搜索。

### 精简版查询模板（推荐，6-10 次搜索）

将多个独立查询合并为更少的综合查询，减少搜索次数。

```
# ===== 第 1 波：综合新闻（3-4 次搜索）=====

# 1. 英文聚合站点 + 综合新闻
"AI news {date_range} site:kite.kagi.com OR site:tldr.tech OR site:venturebeat.com"

# 2. 中文聚合站点 + 综合新闻
"AI 新闻 {date_range} site:news.aibase.com OR site:jiqizhixin.com OR site:qbitai.com"

# 3. 开发者工具更新（合并所有工具为一次搜索）
"Claude Code OR Cursor OR Copilot OR Windsurf OR Devin OR Lovable update OR release {date_range}"

# ===== 中间穿插 fetch 抓取 Changelog 页面（不消耗搜索配额）=====
# 直接 fetch 以下 URL（按需选择）：
#   - https://code.claude.com/docs/en/changelog
#   - https://github.com/anthropics/claude-code/releases        ← GitHub Releases 更结构化，含 highlight
#   - https://www.cursor.com/changelog
#   - https://platform.openai.com/docs/changelog
#   - https://ai.google.dev/changelog                          ← Google AI Studio / Gemini API 更新
#   - https://blog.google/innovation-and-ai/models-and-research/  ← Google 模型发布（Gemini 系列）
#   - https://github.blog/changelog/label/copilot/
#   - https://sdk.vercel.ai/changelog
#   - https://v0.dev/changelog
#
# ⚠️ Changelog 处理注意事项：
#   Changelog 页面通常很长（每个版本几十条变更），容易淹没重要功能。
#   处理策略：fetch changelog 后，重点关注 "Added" 开头的条目（新功能），
#   特别是新增的命令（/xxx）、新模式、新工具——这些对用户影响最大。
#   同时用一次搜索补充社区解读（见下方查询 3b），社区文章会突出最有传播价值的功能。

# 3b. 开发者工具社区解读（补充 changelog，捕捉热点功能）
#    changelog 只列变更，社区文章会突出最有价值的功能（如 /btw 省 50% token）
"Claude Code new feature OR command {date_range}"

# ===== 第 2 波：产品导向技术搜索（2-3 次合并查询）=====

# 4. 视觉 + 3D + 设计方向（合并图像/3D/设计）
"image generation API OR text to 3D OR AI design tool OR SVG generation new release {date_range}"

# 5. Agent + 语音 + 搜索推荐方向（合并交互与后端）
"AI agent framework OR voice AI OR embedding model OR RAG new release {date_range}"

# 6. 竞品动态
"Canva AI OR Figma AI OR Adobe AI new feature {date_range}"

# ===== 中间穿插 fetch 抓取大厂博客（按线索选择性访问）=====
# 优先 fetch 以下大厂博客首页（不消耗搜索配额）：
#   - https://blog.google/innovation-and-ai/models-and-research/  ← Google 模型发布（路径与 /technology/ai/ 不同！）
#   - https://openai.com/news/
#   - https://www.anthropic.com/news
#   - https://ai.meta.com/blog/

# ===== 第 3 波：补充（仅在信息不足时，1-2 次搜索）=====

# 7. 国内大厂动态（如第 1 波中文搜索已覆盖则跳过）
"DeepSeek OR 智谱 OR Qwen OR 豆包 OR 可灵 发布 OR 更新 {date_range}"

# 8. 开源与框架（按需）
"langchain OR llamaindex OR ollama OR vllm release {date_range}"
```
